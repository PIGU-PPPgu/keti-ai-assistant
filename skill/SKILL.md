# 课题 AI 助手 Skill

## 激活条件

用户提到以下任意关键词时激活：
- 课题申报书、申报书、课题申请
- 开题报告、中期检查、结题报告
- 职称评审、课题研究
- 帮我写课题、生成课题

## 工作模式

### 模式 A：直接生成（无服务器）

如果没有配置 `KETI_API_URL`，直接用 AI 生成。

**信息收集流程：**

1. 识别文档类型（申报书/开题/中期/结题）
2. 逐步收集必要参数：
   - 课题级别（区级/市级/省级）
   - 学科（数学/语文/英语/物理/化学/生物等）
   - 学段（小学/初中/高中）
   - 研究方向（核心素养/大单元教学/跨学科融合/数字化教学/AI赋能等）
   - 课题名称（可选，留空自动生成）
3. 展示生成计划，等用户确认
4. 按章节生成完整文档

**生成规范：**
- 字数：区级5000-8000字，市级8000-10000字，省级10000字以上
- 参考文献：至少30篇，格式规范，近3年≥60%
- 不杜撰文献，使用知识库中确认存在的文献
- 创新点具体实在，不夸大

### 模式 B：调用服务（有服务器）

如果配置了 `KETI_API_URL`，通过 API 调用完整的多 Agent 系统：
- TitleAgent：生成高质量课题名
- LiteratureAgent：联网搜索真实文献
- GeneratorAgent：并行章节生成 + Critic 评分循环
- ReviewerAgent：去 AI 化润色

```bash
# 检查服务是否可用
curl $KETI_API_URL/health
```

```
- 课题级别：{level}

- 研究方向：{direction}

五、研究重难点与创新之处

```

## 输出格式

生成完成后：
1. 先输出课题名称和字数统计
2. 输出完整文档（Markdown 格式）
3. 提示用户可以要求修改或导出 Word

如果用户在有服务器的环境（配置了 KETI_API_URL），引导他们访问 Web 界面获得更好体验（实时进度、Word 导出等）。

## 注意事项

- 生成内容较长，分段输出避免超时
- 参考文献必须是真实存在的，不确定的不要写
- 创新点要具体，避免"国内首创"等夸大表述
- 经费预算要符合该级别课题的实际情况
